## 2. ① 技术栈升级（攻"技术难度 30%"）

### 2.1 升级后架构（七层 RAG Pipeline）

```
用户问题
  ↓
[L1 查询改写] HyDE 假设文档生成 + Multi-Query 多视角改写（DeepSeek）
  ↓
[L2 双路召回] BGE-large-zh-v1.5 密集向量(FAISS HNSW) ‖ jieba+BM25 稀疏召回
  ↓
[L3 RRF 融合] Reciprocal Rank Fusion 替代当前加权和（更稳健、无需调参）
  ↓
[L4 Cross-Encoder 重排] BGE-reranker-v2-m3 对 Top-20 精排，取 Top-5
  ↓
[L5 上下文压缩] 截断/去重，控制 prompt token 预算
  ↓
[L6 LLM 生成] DeepSeek-chat + 引用感知 system prompt + 流式输出
  ↓
[L7 答案校验] DeepSeek-reasoner 自检"是否被检索片段支持"（Self-RAG 思想）
  ↓
带引用的回答 + 置信度
```

### 2.2 新增 / 改造模块（在 `campus_rag/` 下）

| 文件 | 性质 | 内容 |
|---|---|---|
| `campus_rag/embeddings.py` | **新增** | 封装 `sentence-transformers` 加载 `BAAI/bge-large-zh-v1.5`；批量 encode + L2 归一化；本地缓存到 `models/bge_cache/`。 |
| `campus_rag/vectorstore.py` | **新增** | FAISS HNSW 索引封装：build / save / load / search；持久化到 `models/faiss_index.bin` 和 `models/chunk_meta.parquet`。 |
| `campus_rag/retriever.py` | **大改** | 保留 `TfidfRetriever`（消融用）；新增 `DenseRetriever`（FAISS）、`Bm25JiebaRetriever`（jieba 分词 + rank_bm25）、`HybridRRFRetriever`（RRF 融合）。 |
| `campus_rag/reranker.py` | **新增** | 封装 `BAAI/bge-reranker-v2-m3` cross-encoder；Top-20 → Top-5。 |
| `campus_rag/query_rewriter.py` | **新增** | `hyde_rewrite()` 用 DeepSeek 生成假设答案再嵌入；`multi_query_expand()` 生成 3 个改写 query。 |
| `campus_rag/splitter.py` | **改造** | 替换为基于 `langchain_text_splitters.RecursiveCharacterTextSplitter` + 中文标点感知；chunk_size=400，overlap=80。 |
| `campus_rag/generator.py` | **大改** | 重写 `llm_answer()`：使用 DeepSeek `deepseek-chat` + 流式 + 引用 system prompt；新增 `self_rag_verify()` 用 `deepseek-reasoner` 校验。 |
| `campus_rag/memory.py` | **新增** | 多轮对话窗口（保留最近 3 轮，用于 Streamlit 聊天界面）。 |
| `campus_rag/evaluate.py` | **大改** | 新增 LLM-as-judge 指标（faithfulness, answer_relevancy, context_precision，RAGAS 思想自实现）；保留 Hit@K/MRR；新增 nDCG@5。 |

### 2.3 依赖更新（`requirements.txt` 合并 advanced）

新增稳定依赖：`sentence-transformers`, `faiss-cpu`, `rank-bm25`, `jieba`, `langchain-text-splitters`, `openai`（已用）, `pyyaml`（配置）, `httpx`（流式）。

### 2.4 必须保留的"创新点"包装话术（写进报告）

1. **HyDE 改写** → "解决问句和文档措辞不一致问题"
2. **RRF 融合** → "免调参的排名融合，比传统加权更鲁棒"
3. **Cross-Encoder 重排** → "两阶段检索：召回求全、重排求精"
4. **Self-RAG 校验** → "用推理模型自检答案的'有据可循'，降低幻觉"
5. **DeepSeek-reasoner LLM-as-judge** → "受 RAGAS 启发的端到端答案质量评估"
