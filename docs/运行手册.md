## 1. 项目简介

### 这是什么？

一个**RAG（检索增强生成）**校园智能问答系统。用户输入校园相关问题（如"图书馆周末几点开门？"），系统从知识库中检索相关文档，结合大模型生成准确回答。

### 核心技术栈（七层管线）

```
用户问题
  ↓
[L1 查询改写] HyDE 假设文档生成（DeepSeek）
  ↓
[L2 双路召回] BGE-large-zh-v1.5 密集向量 + jieba-BM25 稀疏检索
  ↓
[L3 RRF 融合] Reciprocal Rank Fusion 免调参融合
  ↓
[L4 重排序] BGE-reranker-v2-m3 Cross-Encoder 精排
  ↓
[L5 上下文压缩] 截断去重
  ↓
[L6 LLM 生成] DeepSeek-chat 流式输出
  ↓
[L7 答案校验] DeepSeek-reasoner Self-RAG 验证
  ↓
带引用的回答
```

### 知识库规模

| 指标 | 数值 |
|---|---|
| 原始文档 | 179 篇 |
| 文档来源 | 同济新闻、研究生院、教务处、本科招生、体育部、学生事务 + 人工FAQ |
| Chunk 数 | 374 个 |
| 类目数 | 16 个 |

## 2. 环境搭建

### 2.1 前置要求

- Python 3.10 或更高版本
- Windows / macOS / Linux 均可
- 建议 8GB+ 内存（运行 BGE 模型需要）

### 2.2 安装步骤

```bash
# 1. 进入项目目录
cd Python-AI-main/Python-AI-main

# 2. 创建虚拟环境（推荐，避免污染系统 Python）
python -m venv venv
# Windows 激活：
venv\Scripts\activate
# Mac/Linux 激活：
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt
```

### 2.3 配置 DeepSeek API

在项目**上级目录**（`Python-AI-main/`）创建 `deepseek.env` 文件：

```
DEEPSEEK_API_KEY=你的API密钥
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_CHAT_MODEL=deepseek-chat
DEEPSEEK_REASONER_MODEL=deepseek-reasoner
```

> 已有 `deepseek.env` 则跳过。API key 请向凌霄获取。

### 2.4 验证安装

```bash
python -c "import sentence_transformers, faiss, openai; print('环境就绪!')"
```

## 4. 数据流水线

### 4.1 快速开始（已有现成数据）

如果 `data/raw/combined_kb.jsonl` 已存在，直接运行：

```bash
# Step 1: 数据清洗 + 分块（输出 data/processed/chunks.csv）
python scripts/01_prepare_data.py

# Step 2: 构建全部索引（输出 models/ 下各索引文件）
python scripts/02_build_index.py
```

### 4.2 从头爬取数据（如需要）

```bash
# 1. 爬取同济网站（约2-3分钟，110+篇文章）
python scripts/10_crawl_tongji.py

# 2. 清洗爬取页面
python scripts/11_pages_to_kb.py

# 3. 合并爬取数据与人工FAQ（需手动执行以下Python）：
python -c "
import json
from pathlib import Path

def load(f): return [json.loads(l) for l in open(f,'r',encoding='utf-8') if l.strip()]

kb = load('data/raw/campus_kb.jsonl')
faq = load('data/raw/campus_faq.jsonl')
combined = kb + faq
Path('data/raw/combined_kb.jsonl').write_text(
    '\n'.join(json.dumps(x, ensure_ascii=False) for x in combined),
    encoding='utf-8'
)
print(f'合并完成: {len(combined)} 条')
"

# 4. 重新分块 + 建索引
python scripts/01_prepare_data.py
python scripts/02_build_index.py
```

### 4.3 生成评测集（如需要）

```bash
# 基于知识库自动生成评测问题（需要DeepSeek API）
python scripts/12_build_eval.py
```

## 5. 启动应用

### 5.1 Streamlit（推荐主界面）

```bash
streamlit run app.py
```

浏览器打开 `http://localhost:8501`

**功能说明：**
- 在底部输入框输入问题，回车发送

- 点击"清除对话历史"重置对话

### 5.2 Gradio（备用界面）

```bash
python gradio_app.py
```

浏览器打开 `http://localhost:7860`

### 5.3 命令行

```bash
# 基础用法
python cli.py "图书馆开放时间"

# 指定检索策略
python cli.py --strategy rrf "研究生选课流程"

# 开启 Self-RAG 校验
python cli.py --verify "一卡通丢了怎么办"
```

### 6.1 快速消融（仅检索指标，约2分钟）

```bash
python scripts/20_run_ablation.py --quick --skip-hyde --skip-baseline
```

输出 `logs/ablation_results.csv`，包含 Hit@1/3/5、MRR、延迟。

### 6.2 完整消融（含LLM生成 + LLM-as-judge，约30分钟）

```bash
python scripts/20_run_ablation.py --skip-hyde
```

会调用 DeepSeek API 生成回答并评估 Faithfulness、Relevancy 等指标。

### 6.3 当前评测结果

在374 chunks、190题评测集上：

| 实验 | Hit@1 | Hit@3 | MRR | 延迟 |
|---|---|---|---|---|
| TF-IDF | 67.4% | 68.9% | 0.683 | 2.6ms |
| BM25-jieba | 64.2% | 67.9% | 0.661 | 0.6ms |
| BGE Dense | 67.9% | 70.0% | 0.688 | 131ms |
| **RRF Hybrid** | 67.9% | 70.5% | **0.694** | 128ms |
| **+ Reranker** | **70.0%** | **71.6%** | **0.709** | 606ms |

> 说明：指标不再像之前60条玩具数据那样全1.00，现在是诚实的、在真实数据上可复现的结果。

## 8. 常见问题

### Q: 报错 "DEEPSEEK_API_KEY not found"
A: 确认 `deepseek.env` 在项目上级目录且格式正确。检查路径：`Python-AI-main/deepseek.env`

### Q: 首次运行很慢？
A: 首次运行会自动下载 BGE 模型（约1.3GB），请耐心等待。后续会使用缓存。

### Q: Reranker 模式特别慢？
A: Cross-Encoder 在 CPU 上推理约需10秒/查询。日常使用建议选"RAG (RRF only)"模式（80ms）。

### Q: 怎么加更多数据？
A: 两种方式：
1. 在 `data/raw/campus_faq.jsonl` 中追加新FAQ条目
2. 在 `data/source_docs/` 中放入 PDF/DOCX 文件，修改爬虫脚本
然后重新运行 4.1 步骤。

### Q: 评测指标怎么解读？
A:
- **Hit@1/3/5**: 正确答案在检索结果前1/3/5名的比例
- **MRR**: 平均倒数排名，越接近1越好
- **Faithfulness**: 回答是否有据可循（LLM判断）
- **Answer Relevancy**: 回答是否切题

### Q: Streamlit 报错 "No module named 'campus_rag'"
A: 确保在项目根目录（`Python-AI-main/Python-AI-main/`）运行 `streamlit run app.py`
