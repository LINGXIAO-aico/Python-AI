# 测试覆盖与代码质量任务完成说明

本文档用于说明《B07 RAG 校园智能问答助手 — 项目质量提升计划》中 2.1-2.6 部分的完成情况。该部分的核心目标是补齐关键模块测试、提高自动化验证能力、让项目从“能运行”进一步提升为“可验证、可维护、可复现”的工程化项目。

## 一、任务背景

本项目是一个基于 RAG 的校园智能问答助手，核心流程包括查询改写、向量编码、FAISS 向量检索、BM25 关键词召回、RRF 融合、Cross-Encoder 重排序、DeepSeek 生成和 Self-RAG 校验。

在质量提升计划中，2.1-2.6 目标是：

- 补充 `embeddings.py`、`vectorstore.py`、`reranker.py`、`query_rewriter.py` 的直接测试；
- 确保代码风格检查、类型检查和离线测试通过；
- 补充 coverage 配置并把测试覆盖率提升到 80% 以上。

这部分工作并不是新增 UI 或改问答效果，而是增强项目的工程可信度。它能证明项目中关键 RAG 组件不是只停留在代码层面，而是有自动测试持续验证。

### 实际完成

测试分为两类：

- query 编码会自动加上“为这个句子生成表示以用于检索相关文章：”前缀；

第二类是真实模型集成测试，使用 `@pytest.mark.model` 标记。该测试会在本地存在 BGE 模型缓存时检查真实输出是否为 1024 维，并验证向量归一化。

## 九、额外发现并修复的问题

本次补测试时发现 `campus_rag/generator.py` 中存在一个真实 bug：

Self-RAG 校验提示词 `VERIFY_PROMPT` 里包含 JSON 示例，但花括号没有转义。由于后续使用 `.format(context=..., answer=...)` 填充模板，Python 会把 JSON 示例里的 `{}` 当作格式化占位符，导致实际调用时报 `KeyError`。

已修复为双花括号形式：

```python
{{
  "claims": [{{"text": "主张内容", "status": "supported|contradicted|unsupported"}}],
  "verdict": "fully_supported|partially_supported|unsupported",
  "explanation": "一句话解释"
}}
```

该修复提升了 Self-RAG 校验模块的实际可用性。
